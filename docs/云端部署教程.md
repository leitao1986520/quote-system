# 云端部署教程（腾讯云 CloudBase：国内直连，数据持久）

本教程面向零基础用户。部署到**腾讯云 CloudBase** 后：
- 前端网页托管在 CloudBase 静态网站（国内域名，手机/电脑直接打开，不被墙）
- 后端运行在 CloudBase 云函数
- 数据保存在 **CloudBase 云数据库**，重启/重新部署都不丢失

> 之前 Vercel / Render 的 `*.vercel.app` / `render.com` 域名在国内被墙，故改用 CloudBase。

---

## 第一步：准备

1. 注册腾讯云账号：https://cloud.tencent.com （微信扫码即可）
2. 开通 CloudBase：https://console.cloud.tencent.com/tcb
   - 新建环境，选**按量计费（有免费额度）**或**基础版 1 元体验版**
   - 记住环境 ID，形如 `quote-system-1a2b3c`（接下来要用）
3. 在「云数据库」中创建以下 5 个集合（集合名一字不差）：
   - `users`
   - `inquiries`
   - `inquiry_items`
   - `quotations`
   - `quotation_items`

> 集合创建方式：CloudBase 控制台 → 你的环境 → 数据库 → 新建集合，逐个添加即可（无需建字段，代码会自动写入）。

---

## 第二步：在本地用 CLI 部署（推荐，一条命令）

> 本机若没有 node/npm，建议在一台能正常 `npm install` 的电脑上执行（公司安全软件拦截 esbuild 不影响 CloudBase CLI 与后端，但前端构建需要干净环境；也可直接在 CloudBase 控制台的「静态网站托管」手动上传 `web/dist`）。

1. 安装 CloudBase CLI：
   ```bash
   npm i -g @cloudbase/cli
   ```
2. 登录（会弹出浏览器授权）：
   ```bash
   tcb login
   ```
3. 进入项目根目录，编辑 `cloudbaserc.json`，把 `{{ENV_ID}}` 改成你的环境 ID：
   ```json
   "envId": "quote-system-1a2b3c"
   ```
4. 构建前端（产出 `web/dist`）：
   ```bash
   npm run build
   ```
5. 一键部署（静态网站 + 云函数）：
   ```bash
   tcb framework deploy
   ```
   - 若只想分别部署：
     ```bash
     tcb hosting deploy web/dist -e 你的环境ID
     tcb fn deploy quote-api -e 你的环境ID
     ```

---

## 第三步：配置云函数环境变量

CloudBase 控制台 → 你的环境 → 云函数 → `quote-api` → 配置 → 环境变量，添加两条：

| 键 | 值 |
|---|---|
| `JWT_SECRET` | `12d2b37ff2c00c43fbf1e09ebe0d7d010555df5729af7ac41ece00aa6e308801` |
| `TCB_ENV` | 你的环境 ID（如 `quote-system-1a2b3c`） |

保存后**重启一次云函数**使其生效。

---

## 第四步：配置静态网站路由（SPA 回退 + /api 转发）

CloudBase 控制台 → 你的环境 → 静态网站托管 → 基础设置 → 路由规则，确保类似如下（已写在 `cloudbaserc.json` 的 `hosting.rewrite` 中，CLI 部署会自动应用；手动部署则需手动加）：

- 来源 `/api/**` → 转发到云函数 `quote-api`
- 来源 `/api` → 转发到云函数 `quote-api`
- 来源 `**` → 重写到 `/index.html`（保证前端刷新不 404）

---

## 验证后端

打开 `https://你的静态网站域名/api/health`，应返回 `{"ok":true}`。

## 常见问题

**Q：打不开静态网站域名？**
- 确认静态网站托管已开启（控制台 → 静态网站托管 → 开启）
- 确认 `web/dist` 已成功上传（含 `index.html`）
- 手机用流量、电脑用浏览器分别试；CloudBase 国内域名一般直连可达

**Q：前端能开，但登录报 500 / /api 不通？**
- 检查云函数 `quote-api` 是否已部署、环境变量 `TCB_ENV` 是否正确
- 看云函数日志（控制台 → 云函数 → 日志）里的报错，通常是云数据库权限或集合未创建

**Q：改了代码怎么更新？**
- 重新 `npm run build`，再 `tcb hosting deploy web/dist`；后端改动后 `tcb fn deploy quote-api`
- 若代码已推到 GitHub，也可在 CI 中执行同样命令

**Q：数据安全吗？**
- 数据存在 CloudBase 云数据库，持久保存，不会因函数重启丢失。正式使用建议配置数据库权限规则和定期备份。

**Q：免费额度够用吗？**
- 个人/小团队试用足够。云函数按调用量计费、静态网站有免费流量，云数据库按存储/读写计费，基础版体验价低。
