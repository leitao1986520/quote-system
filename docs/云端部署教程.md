# 云端免费部署教程（Vercel 单平台：前端 + 后端一体）

本教程面向零基础用户，全程免费，只需一个平台（Vercel）。部署完成后：
- 前端网页 + 后端 API 都运行在 Vercel
- 手机、电脑都能通过 Vercel 提供的网址访问

> 部署在云端时，你本机 esbuild 被安全软件拦截的问题**不存在**（云端是干净的 Linux 环境，自动构建）。

## 第二步：部署到 Vercel（免费，前后端一体）

1. 打开 https://vercel.com ，用 **GitHub 账号登录**（已注册）
2. 点 **Add New → Project**，导入仓库 `leitao1986520/quote-system`
3. 配置项目（关键几项正确即可，其余默认）：
   - **Framework Preset**：选 **Other**
   - **Root Directory**：选 **`.`**（仓库根，不要选 web 或 server 子目录）
   - **Build Command**：`npm run build`（已写在根 `package.json`）
   - **Install Command**：`npm run install:server`（已写）
   - **Output Directory**：`web/dist`（已写）
4. 展开 **Environment Variables**，添加一条：
   - `JWT_SECRET` = `12d2b37ff2c00c43fbf1e09ebe0d7d010555df5729af7ac41ece00aa6e308801`
5. 点 **Deploy**，等待 1～3 分钟构建完成
6. Vercel 会给你一个网址，形如 `https://quote-system-xxx.vercel.app`

> 部署成功后，前端页面和 `/api/*` 接口都由 Vercel 提供，同源访问、无需跨域配置。

## ⚠️ 数据持久化说明（重要）

Vercel 的运行环境是**临时**的，SQLite 数据库写在 `/tmp`，**函数重新部署或长时间冷启动后，数据会重置**（演示数据会自动重新生成）。
- 适合：演示、试用、展示
- 不适合：需要长期保存业务数据的正式生产
- 若需持久化：后续把数据库换成 Vercel Postgres / Supabase 等托管数据库即可

## 常见问题

**Q：部署后打开网页空白 / 接口 404？**
- 确认 Root Directory 选的是 `.`（仓库根），不是 web 子目录
- 访问 `/api/health` 看是否返回 ok，判断后端函数是否正常

**Q：改了代码怎么更新？**
- 直接 `git push` 到 GitHub，Vercel 会**自动重新部署**。

**Q：免费额度够用吗？**
- 个人/小团队试用完全够。Vercel 免费层带宽 100GB/月、Serverless 函数有每日调用与执行时长额度，演示足够。

**Q：数据安全吗？**
- 当前为演示级 SQLite（临时磁盘）。正式长期使用建议升级到托管数据库。
