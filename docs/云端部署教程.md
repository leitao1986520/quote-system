# 云端免费部署教程（Render 后端 + Vercel 前端）

本教程面向零基础用户，全程免费。部署完成后：
- 后端（API + 数据库）运行在 Render 免费层
- 前端（网页）运行在 Vercel 免费层
- 手机、电脑都能通过 Vercel 提供的网址访问

> 部署在云端时，你本机 esbuild 被安全软件拦截的问题**不存在**（云端是干净的 Linux 环境，自动构建）。

---

## 第一步：把代码推到 Git 仓库

云端部署平台都需要从一个 Git 仓库拉代码。推荐用 GitHub（免费）。

1. 注册 GitHub：https://github.com
2. 新建一个仓库，例如 `quote-system`（选 Public 或 Private 均可）
3. 把 `quote-system/` 整个目录推上去：
   ```bash
   cd quote-system
   git init
   git add .
   git commit -m "init"
   git branch -M main
   git remote add origin https://github.com/你的用户名/quote-system.git
   git push -u origin main
   ```

---

## 第二步：部署后端到 Render（免费）

1. 打开 https://render.com ，用 GitHub 账号注册/登录
2. 点 **New + → Blueprints**（或 New → Web Service）
   - 如果选 Blueprints：连接仓库后会自动读取 `render.yaml`，按提示确认即可
   - 如果手动建 Web Service，按下面填：
     - **Root Directory**：`quote-system/server`
     - **Build Command**：`npm install`
     - **Start Command**：`npm start`
     - **Instance Type**：`Free`
3. 在 **Environment** 里添加变量：
   - `NODE_ENV` = `production`
   - `JWT_SECRET` = 随便一段长随机字符串（如 `openssl rand -hex 32` 的结果；也可留空用默认值）
4. 点击 Create，等待 1～2 分钟构建完成
5. 记下 Render 给你的后端地址，形如 `https://quote-system-backend.onrender.com`
   - 在浏览器访问 `https://你的后端地址/api/health` ，应返回 `{"ok":true}`

> 注意：Render 免费层 15 分钟无访问会自动休眠，首次访问可能慢几秒，属正常现象。

### （可选）让数据库不丢数据
Render 免费层重启后磁盘会重置，演示数据会丢失（不影响功能，只是要重新注册/录入）。
若要持久化：在 Render 控制台给该服务挂一个 **Disk**（挂载路径 `/var/data`），然后在 Environment 加
`DATA_DIR` = `/var/data` 即可。

---

## 第三步：部署前端到 Vercel（免费）

1. 打开 https://vercel.com ，用 GitHub 账号注册/登录
2. 点 **Add New → Project**，导入刚才的 GitHub 仓库
3. 配置：
   - **Root Directory（根目录）**：选择 `quote-system/web`
   - **Framework Preset**：选 `Vite`（Vercel 会自动识别）
   - **Build Command**：`npm run build`（默认即是）
   - **Output Directory**：`dist`（默认即是）
4. 在 **Environment Variables** 中添加：
   - `VITE_API_BASE` = `https://你的后端地址`（即第二步记下的 Render 地址，**不要带结尾斜杠**）
   - 例如：`https://quote-system-backend.onrender.com`
5. 点击 Deploy，等待构建完成（约 1 分钟）
6. Vercel 会给你一个网址，形如 `https://quote-system-xxx.vercel.app`
   - 首次部署后，还要到 **Settings → Environment Variables** 确认 `BACKEND_URL` 已设置（见下）

### 关于 vercel.json 里的 BACKEND_URL
项目根 `web/vercel.json` 已配置把 `/api/*` 转发到 `${BACKEND_URL}`。请在 Vercel 项目
**Settings → Environment Variables** 里添加：
- `BACKEND_URL` = `https://你的后端地址`（和 `VITE_API_BASE` 填一样的值）

> 如果你不想用 vercel.json 代理，也可以直接把 `VITE_API_BASE` 配好，前端会直连后端（跨域已通过后端 CORS 放行）。两种方式任选其一。

## 常见问题

**Q：部署后打开网页空白 / 接口 404？**
- 检查 Vercel 的 `VITE_API_BASE` / `BACKEND_URL` 是否填对（必须是 Render 真实地址，带 `https://`）
- 确认 Render 后端已启动（访问 `/api/health` 能返回 ok）

**Q：改了代码怎么更新？**
- 直接 `git push` 到 GitHub，Render 和 Vercel 都会**自动重新部署**。

**Q：免费额度够用吗？**
- 个人/小团队试用完全够。Vercel 免费层带宽 100GB/月；Render 免费层 750 小时/月（约 31 天），足够一个常驻小服务。

**Q：数据安全吗？**
- 免费层数据库是 SQLite 单文件，存在 Render 服务磁盘上。重要生产数据建议后续升级到云数据库（PostgreSQL/MySQL）。本教程定位为免费可用方案。
